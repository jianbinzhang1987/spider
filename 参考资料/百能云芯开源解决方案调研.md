# 百能云芯开源解决方案调研

调研日期：2026-06-24

## 一、是否存在百能云芯专用开源方案

结论：**未找到可直接复用的百能云芯 icdeal.com 专用开源爬虫/adapter**。

已检索关键词：

- `icdeal`
- `icdeal.com`
- `searchapi.icdeal.com`
- `百能云芯`
- `百能芯城`
- `百能云芯 爬虫`

GitHub repository search 结果：

- `icdeal` 仅命中 1 个无关仓库 `tammat11/icdealsonly`。
- `icdeal.com` 无结果。
- `searchapi.icdeal.com` 无结果。
- `百能云芯` / `百能芯城` 命中的高星仓库为书籍收藏类仓库，非爬虫实现。

因此，百能云芯不能按“找一个现成开源 adapter 接入”的方式推进，只能基于通用开源组件自研。

## 二、可复用的开源组件

### 2.1 Playwright / playwright-stealth

用途：

- 使用真实浏览器执行 JS。
- 降低普通 headless 浏览器特征。
- 适合做人工验证后的会话复用、页面渲染、接口捕获。

代表项目：

- `AtuboDad/playwright_stealth`
- PyPI `playwright-stealth`

适配百能云芯的判断：

- 可以改善基础自动化特征。
- 不能指望单独绕过百能 WAF 滑块。
- 适合作为 BrowserAdapter 基础能力，不应作为“破解 WAF”的核心依赖。

### 2.2 curl_cffi

用途：

- 模拟 Chrome/Safari 的 TLS/JA3/HTTP2 指纹。
- 替代 `requests` / `httpx` 处理部分因 TLS 指纹被拒的站点。

代表项目：

- `lexiforest/curl_cffi`

适配百能云芯的判断：

- 可用于访问公开 API，如热搜接口：

```text
https://searchapi.icdeal.com/api/v1/sku_search/get_search_keywords?day=7
```

- 但百能真实搜索入口会跳 WAF 滑块，`curl_cffi` 无法解决页面级人机验证。
- 可作为辅助 HTTP 客户端，不是主方案。

### 2.3 ddddocr / 滑块识别类项目

用途：

- 文字验证码识别。
- 部分滑块缺口识别。

代表项目：

- `sml2h3/ddddocr` / `86maid/ddddocr`
- `ddddocr-node`

适配百能云芯的判断：

- 技术上可能用于识别滑块，但百能 WAF 通常还会校验行为轨迹、环境指纹、Cookie、时间戳和服务端风控。
- 自动化滑块会带来稳定性、合规性和账号风险。
- 不建议作为默认交付方案。最多作为内部实验，不进入生产主流程。

### 2.4 第三方 Web Unlocker / Scraper API

用途：

- 由商业服务处理代理、浏览器渲染、验证码和反爬。

代表类型：

- Bright Data / Oxylabs / ScrapFly 等。

适配百能云芯的判断：

- 可能能降低技术投入，但成本高。
- 国内 B2B 站点命中率不确定。
- 涉及账号、价格、库存等业务数据时，合规和数据泄露风险需要先评估。

## 三、推荐方案排序

### 方案 A：官方/商务接口优先

优先级最高。

百能云芯官网公开有联系方式、BOM 采购和供应链服务介绍。对采购比价系统而言，最稳妥路径是商务沟通：

- 申请商品搜索、库存、价格数据接口。
- 或以 BOM 报价方式对接。
- 或获得白名单/企业账号权限。

优点：

- 稳定。
- 合规。
- 不受 WAF 影响。
- 可获得更完整的阶梯价、库存、货期。

缺点：

- 需要商务沟通。
- 可能需要企业资质或采购量。

### 方案 B：人工验证 + 持久化 Playwright 会话

推荐作为技术兜底方案。

流程：

1. Playwright 打开百能云芯首页或搜索页。
2. 如果跳转到 `waf.icdeal.com/waf/verification`，暂停并提示人工完成滑块。
3. 验证通过后保存 `storage_state`。
4. 后续查询复用同一个 context 和 Cookie。
5. 设置低并发、低频率、缓存和失败熔断。

当前代码已按该方案推进：

```text
python src\main.py STM32F103C8T6 icdeal
```

当任务包含 `icdeal` 时，主流程会默认启动可见浏览器。若页面进入 `waf.icdeal.com/waf/verification`，程序会等待用户手动完成滑块验证，验证通过后继续访问目标搜索页并解析结果。验证态会保存到：

```text
data/sessions/icdeal.json
```

如果需要强制无头运行，可设置：

```text
BROWSER_HEADLESS=1
```

这种情况下遇到 WAF 会快速失败，不会在不可见浏览器里长时间等待。

优点：

- 不需要破解验证码。
- 实现成本可控。
- 与现有 BrowserAdapter 架构兼容。

缺点：

- 不是完全无人值守。
- Cookie 可能过期。
- WAF 策略变化时仍会失效。

### 方案 C：Playwright + 网络响应捕获

作为方案 B 的增强。

人工验证通过后，不建议直接正则解析整页 HTML，而应：

- 捕获搜索页 XHR / fetch。
- 找到商品列表 JSON。
- 优先解析结构化响应。
- DOM 解析只作兜底。

当前问题：

- 未通过 WAF 前，真实商品搜索接口不可见。
- 需要在人工验证后的浏览器会话里继续抓包确认。

### 方案 D：自动识别滑块

不推荐作为生产方案。

虽然有 `ddddocr` 等开源组件，但百能 WAF 属于站点安全防护，不是普通图形验证码。自动化识别存在：

- 成功率不稳定。
- 触发账号/IP 风控。
- 后续维护成本高。
- 合规风险高。

只能作为内部技术验证，不建议承诺交付。

## 四、对当前代码的建议

当前文件：`src/adapters/icdeal.py`

建议调整：

1. 不再把百能云芯标记为“Playwright 可绕过 WAF”。
2. 增加明确 WAF 检测：
   - 当前 URL 包含 `waf.icdeal.com/waf/verification`
   - 页面文本包含 `正在验证您的身份`
   - 页面文本包含 `滑动验证码`
3. 遇到 WAF 时返回 `FAILED`，错误信息为：

```text
WAF滑块验证，需要人工验证或官方接口
```

4. 增加可选参数：
   - `manual_verify=True`
   - `storage_state_path=data/sessions/icdeal.json`
5. 如果已有可用 `storage_state`，优先复用。
6. 查询频率建议：
   - 单账号串行或低并发。
   - 每个型号间隔 3-8 秒。
   - 对同型号结果做 24 小时缓存。

## 五、最终建议

百能云芯的开源路径不是“找专用爬虫”，而是：

1. **先走官方/商务接口**。
2. 技术兜底采用 **Playwright + 人工验证 + 会话持久化**。
3. 验证通过后再做 **XHR 捕获和结构化解析**。
4. 不建议把自动滑块识别作为生产能力承诺。

如果项目需要近期可交付，百能云芯应放在“人工辅助/高风险站点”分组，不应和京东工品汇、艾汐芯城这类已有可用接口链路的站点同等排期。
