# 各网站技术方案调研与策略分析

## 总览

根据需求文档，共涉及 **18 个网站**（国际 3 + 国内 15）。以下逐一分析每个网站的最佳抓取策略。

---

## 一、国际站点（3 个）

### 1. Digi-Key (www.digikey.com)

| 项目 | 详情 |
|------|------|
| **推荐策略** | **官方 API（强烈推荐）** |
| **API 文档** | https://developer.digikey.com/ |
| **API 版本** | Product Information V4 |
| **认证方式** | OAuth 2.0（支持 2-Legged 和 3-Legged） |
| **关键接口** | `KeywordSearch`、`ProductDetails`、`ProductPricing`、`PricingOptionsByQuantity` |
| **数据格式** | JSON |
| **限制** | 需注册开发者账号，申请 Client ID + Secret |
| **Python 支持** | 已有第三方库 `digikey_apiv4`（GitHub: tapegoji/digikey_apiv4） |
| **返回数据** | 型号、品牌、阶梯价格、库存、货期、产品 URL 等完整信息 |
| **反爬情况** | API 无反爬，浏览器方式有 Cloudflare + DataDome |

**结论**：Digi-Key 必须用 API，浏览器方式几乎不可行（当前实现失败率极高）。API 可直接获取阶梯价格和库存，完美满足需求。

**API 调用示例**（伪代码）：
```python
# GET /products/v4/search/{keyword}/productdetails
# Headers: X-DIGIKEY-Client-Id, Authorization: Bearer {token}
# 返回：价格梯度、库存、货期、datasheet URL 等
```

### 2. Mouser (www.mouser.com)

| 项目 | 详情 |
|------|------|
| **推荐策略** | **官方 API（强烈推荐）** |
| **API 文档** | https://api.mouser.com/api/docs/ui/index |
| **API 版本** | V2 |
| **认证方式** | API Key（query parameter） |
| **申请地址** | https://www.mouser.com/api-hub/ |
| **关键接口** | `/search/keyword`、`/search/partnumber`、`/search/partnumberandmanufacturer` |
| **数据格式** | JSON |
| **限制** | 免费申请，有每日调用次数限制 |
| **Python 支持** | 直接 httpx/requests POST 即可 |
| **返回数据** | ManufacturerPartNumber、PriceBreaks、Availability、LeadTime、ProductDetailUrl |

**结论**：Mouser API 申请简单（免费），当前项目代码中已有 API 调用逻辑框架（`mouser.py` 的 `_search_via_api`），只需补上 API Key 即可。

**API 调用示例**：
```python
POST https://api.mouser.com/api/v2/search/keyword?apiKey={key}
Body: {"SearchByKeywordRequest": {"keyword": "RC0402FR-0710KL", "records": 10}}
# 返回：Parts[].PriceBreaks, Parts[].Availability, Parts[].LeadTime
```

### 3. element14 / Farnell / Newark (www.element14.com)

| 项目 | 详情 |
|------|------|
| **推荐策略** | **官方 API（推荐）** |
| **API 文档** | https://partner.element14.com/docs |
| **认证方式** | API Key（URL query parameter: `userinfo.apiKey`） |
| **关键接口** | Keyword Search、Manufacturer Part Number Search、Product ID Search |

| **限制** | 需注册账号申请 API Key（24字符），免费 |

**API 调用示例**：
```
GET https://api.element14.com/catalog/products?term=mfpSearch:RC0402FR-0710KL&storeInfo.id=cn.element14.com&userinfo.apiKey={key}&callInfo.responseDataFormat=JSON
```

## 二、国内站点（15 个）

### 4. 立创商城 (www.szlcsc.com)

| 项目 | 详情 |
|------|------|
| **推荐策略** | **官方 API（最佳）/ 内部 HTTP 接口（备选）** |
| **官方 API** | LCSC 开放平台（https://www.lcsc.com/docs/openapi/） |
| **认证方式** | API Key + Signature（HMAC-SHA256） |
| **关键接口** | `getKeyword Search List API`、`getItem Details API` |
| **申请条件** | 需要 LCSC 账号，提交申请材料 |
| **限制** | 1000次/天，200次/分钟（可申请提高） |
| **内部接口** | 立创前端有 XHR 搜索接口（参考 kicad-jlcpcb-tools 开源项目的实现） |
| **返回数据** | 型号、品牌、库存、阶梯价格、最低价 |
| **反爬情况** | 有验证码机制，API 方式无此问题 |

**结论**：优先申请官方 API（LCSC 开放平台），备选方案是分析前端 XHR 接口用 HTTP 直接调用。当前 Playwright 方案失败率高。

## 五、每个网站需要确认的关键信息

在正式开发前，建议对每个目标网站做以下验证：

1. **搜索接口确认**：用浏览器 DevTools (Network tab) 抓取搜索请求，确认是否有 JSON API
2. **登录依赖确认**：搜索和查看价格是否需要登录
3. **阶梯价格位置**：价格是在搜索结果页展示还是需要进入详情页
4. **反爬测试**：用 httpx 发送搜索请求，看是否被拦截
5. **数据格式**：确认返回的价格、库存、货期的 HTML/JSON 结构

## 六、技术栈建议

```
数据获取层：
├── API 客户端（httpx async）
│   ├── DigiKey OAuth2 Client
│   ├── Mouser API Client  
│   ├── element14 API Client
│   ├── LCSC API Client
│   └── Ickey API Client
├── HTTP 爬虫（httpx async + parsel/bs4）
│   ├── 通用搜索解析器
│   └── 各站点适配器（选择器配置）
└── Playwright 兜底（仅京东等强反爬站点）

并发控制：
├── asyncio + Semaphore（每站点独立限流）
├── 自动重试 + 指数退避
└── 熔断器（失败超过阈值则跳过）

数据层：
├── SQLite / JSON 缓存
├── 汇率缓存（12h）
└── 结果聚合 + 最低价比较

输出层：
├── Excel 输出（openpyxl）
└── Streamlit UI
```
