## 二、核心基类设计

### 2.1 BaseSupplierAdapter（直接复用BOM1设计）

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from enum import StrEnum

class SearchType(StrEnum):
    MPN = "mpn"      # 厂商型号
    SKU = "sku"      # 商城编号

class QueryStatus(StrEnum):
    SUCCESS = "success"
    NOT_FOUND = "not_found"
    FAILED = "failed"

@dataclass
class PartResult:
    supplier: str
    query: str
    status: QueryStatus
    mpn: str | None = None
    brand: str | None = None
    package: str | None = None
    description: str | None = None
    stock: int | None = None
    moq: int | None = None
    price_unit: float | None = None
    price_breaks: list[PriceBreak] = field(default_factory=list)
    product_url: str | None = None
    datasheet_url: str | None = None
    error_message: str | None = None

class BaseSupplierAdapter(ABC):
    supplier_name: str

    @abstractmethod
    async def search_by_mpn(self, mpn: str) -> PartResult: ...

    @abstractmethod
    async def close(self) -> None: ...
```

### 2.2 HttpAdapter（curl_cffi 基类）

```python
from curl_cffi.requests import AsyncSession

class HttpAdapter(BaseSupplierAdapter):
    """适用于: 唯样（curl_cffi绕过TLS检测）"""

    def __init__(self, timeout=15.0, min_interval=1.0):
        self._client: AsyncSession | None = None
        self._rate_limiter = AsyncRateLimiter(min_interval)

    def _get_client(self) -> AsyncSession:
        if self._client is None:
            self._client = AsyncSession(
                impersonate="chrome124",  # TLS指纹模拟
                timeout=self.timeout,
                headers=self._default_headers(),
            )
        return self._client

    async def _fetch(self, url, **kwargs):
        await self._rate_limiter.wait()
        client = self._get_client()
        return await client.get(url, **kwargs)
```

### 2.3 BrowserAdapter（Playwright 基类）

```python
from playwright.async_api import Browser, Page

class BrowserAdapter(BaseSupplierAdapter):
    """适用于: 华强、万联、小猫、ICGOO、硬之城、猎芯、百能、艾汐、IC交易网"""

    def __init__(self, browser_pool: BrowserPool):
        self._pool = browser_pool

    async def _new_page(self) -> Page:
        return await self._pool.acquire_page()

    async def _search_with_browser(self, url: str, wait_selector: str = None) -> str:
        page = await self._new_page()
        try:
            await page.goto(url, timeout=30000)
            if wait_selector:
                await page.wait_for_selector(wait_selector, timeout=15000)
            else:
                await page.wait_for_timeout(8000)
            return await page.content()
        finally:
            await self._pool.release_page(page)
```

## 三、10个站点具体实现策略

### 3.1 唯样商城 (oneyac.com) — curl_cffi + Token逆向

**验证结论**: curl_cffi 成功绕过 TLS 检测，获取到含型号的 HTML (90KB)。价格数据需调用 soic.oneyac.com JSONP API，但需客户端生成的 token。

**实现策略**:
```python
class OneyacAdapter(HttpAdapter):
    """
    策略: curl_cffi获取搜索页HTML → 解析服务端渲染的基础产品数据
    进阶: 逆向JS中的token生成算法 → 调用soic API获取价格
    """
    search_url = "https://www.oneyac.com/search.html"

    async def search_by_mpn(self, mpn: str) -> PartResult:
        # Step 1: 获取搜索页（curl_cffi已验证可绕过TLS）
        resp = await self._fetch(self.search_url, params={"keyword": mpn})

        # Step 2: 从HTML中提取内嵌的JSON产品数据
        # 页面中有 var xxx = {...} 格式的初始数据
        data = self._extract_embedded_json(resp.text)

        # Step 3（可选）: 生成token调用soic API获取价格
        # token = self._generate_token(mpn)
        # price_data = await self._fetch_soic_api(mpn, token)

        return self._build_result(mpn, data)
```

**技术难点**: token 生成算法逆向（JS混淆代码解析）
**备选方案**: 如果 token 逆向困难，退回 Playwright 方式获取完整数据

### 3.5 ICGOO (icgoo.net) — Playwright + API拦截

**验证结论**: Vue SPA，浏览器渲染后型号可见。API调用在浏览器Session内有效：
- `v8back.icgoo.net/api/search/suggestions/?q={keyword}` → 返回匹配型号
- `v8back.icgoo.net/api/search/supplier/{partno}/1/` → 返回供应商数据

**实现策略**:
```python
class IcgooAdapter(BrowserAdapter):
    """
    策略: Playwright渲染SPA → 拦截API响应获取JSON数据
    关键API: /api/search/supplier/{partno}/1/ (在浏览器内返回JSON)
    """
    async def search_by_mpn(self, mpn: str) -> PartResult:
        page = await self._new_page()
        try:
            # 设置API响应拦截
            api_data = {}
            async def capture_api(response):
                if 'search/supplier' in response.url:
                    try:
                        api_data['supplier'] = await response.json()
                    except: pass

            page.on('response', capture_api)
            await page.goto(f"https://www.icgoo.net/search/{mpn}/1", timeout=30000)
            await page.wait_for_timeout(12000)

            if api_data.get('supplier'):
                return self._parse_api_response(mpn, api_data['supplier'])

            # Fallback: 从渲染DOM提取
            content = await page.content()
            return self._parse_dom(mpn, content)
        finally:
            await self._pool.release_page(page)
```

**复杂度**: 中 — 需要 API 响应拦截

### 3.6 硬之城 (allchips.com) — Playwright + 验证码破解

**验证结论**: 即使 Playwright 也弹验证码（云之锁 CaptchaButton，mode=20 滑块 / mode=10 点击）。

**实现策略**:
```python
class AllchipsAdapter(BrowserAdapter):
    """
    策略: Playwright + ddddocr滑块识别
    验证码: 云之锁(cdn.yzcstatic.com) CaptchaButton
    模式: mode=20(滑块80%), mode=10(点击20%)
    """
    def __init__(self, browser_pool, captcha_solver: CaptchaSolver):
        super().__init__(browser_pool)
        self._solver = captcha_solver

    async def search_by_mpn(self, mpn: str) -> PartResult:
        page = await self._new_page()
        try:
            await page.goto(f"https://www.allchips.com/search?keyword={mpn}", timeout=20000)
            await page.wait_for_timeout(3000)

            # 检测验证码
            if await self._has_captcha(page):
                solved = await self._solve_captcha(page)
                if not solved:
                    return self.failed_result(mpn, "验证码破解失败")
                await page.wait_for_timeout(5000)

            content = await page.content()
            return self._parse_results(mpn, content)
        finally:
            await self._pool.release_page(page)

    async def _solve_captcha(self, page: Page) -> bool:
        """
        云之锁滑块验证码破解流程:
        1. 点击"点击验证"按钮触发验证码
        2. 等待滑块图片加载
        3. 截图 → ddddocr检测缺口位置
        4. 模拟人类滑动轨迹
        """
        # 点击触发
        await page.click('a[onclick="showCap()"]')
        await page.wait_for_timeout(2000)

        # 截图验证码区域
        captcha_img = await page.screenshot()
        gap_x = self._solver.detect_slide_gap(captcha_img)

        # 模拟滑动
        await self._simulate_slide(page, gap_x)
        await page.wait_for_timeout(2000)

        # 验证是否通过
        content = await page.content()
        return 'captcha' not in content.lower()
```

**复杂度**: 高 — 需要验证码识别 + 轨迹模拟
**成功率预估**: 60-80%（ddddocr本地），90%+（第三方打码平台）

### 3.7 猎芯网 (ichunt.com) — Playwright + SPA深度交互

**验证结论**: 纯 SPA 架构，搜索功能通过 JS 渲染。已知 API 域名 `api.ichunt.com` 和 `apibom.ichunt.com` 但未找到正确的搜索端点。

**实现策略**:

```python
class IchuntAdapter(BrowserAdapter):
    """
    策略: Playwright渲染SPA → 模拟搜索交互 → 拦截API响应
    备注: 搜索可能通过hash路由或POST API完成
    """
    async def search_by_mpn(self, mpn: str) -> PartResult:
        page = await self._new_page()
        try:
            # 拦截所有API调用
            api_responses = []
            async def capture(response):
                if 'api.ichunt.com' in response.url or 'apibom' in response.url:
                    api_responses.append({
                        'url': response.url,
                        'status': response.status
                    })
            page.on('response', capture)

            await page.goto("https://www.ichunt.com/", timeout=20000)
            await page.wait_for_timeout(3000)

            # 模拟搜索操作（输入+回车或点击按钮）
            search_input = await page.query_selector('input[type="text"]')
            if search_input:
                await search_input.fill(mpn)
                await search_input.press('Enter')
                await page.wait_for_timeout(8000)

            # 从拦截到的API响应提取数据
            # 或从渲染后的DOM提取
            content = await page.content()
            return self._parse_results(mpn, content, api_responses)
        finally:
            await self._pool.release_page(page)
```

### 3.10 IC交易网 (ic.net.cn) — Playwright + 登录态维持

**验证结论**: 搜索强制重定向到登录页面。

**实现策略**:
```python
class IcnetAdapter(BrowserAdapter):
    """
    策略: 注册账号 → Playwright登录 → 维持Cookie → 搜索
    登录URL: https://member.ic.net.cn/login.php
    """
    def __init__(self, browser_pool, username: str, password: str):
        super().__init__(browser_pool)
        self._username = username
        self._password = password
        self._logged_in = False

    async def _ensure_login(self, page: Page):
        if self._logged_in:
            return
        await page.goto("https://member.ic.net.cn/login.php")
        await page.fill('input[name="username"]', self._username)
        await page.fill('input[name="password"]', self._password)
        await page.click('button[type="submit"]')
        await page.wait_for_timeout(3000)
        self._logged_in = True

    async def search_by_mpn(self, mpn: str) -> PartResult:
        page = await self._new_page()
        try:
            await self._ensure_login(page)
            await page.goto(f"https://www.ic.net.cn/search.php?q={mpn}")
            await page.wait_for_timeout(5000)
            content = await page.content()
            return self._parse_results(mpn, content)
        finally:
            await self._pool.release_page(page)
```

**前置条件**: 需注册 ic.net.cn 账号

## 四、浏览器池设计

```python
class BrowserPool:
    """
    管理Playwright浏览器实例池，复用Context避免重复启动开销
    - 支持并发控制（Semaphore）
    - 支持Context轮换（避免行为画像）
    - 支持Cookie持久化（登录态）
    """
    def __init__(self, max_pages=5, headless=True):
        self._browser: Browser | None = None
        self._semaphore = asyncio.Semaphore(max_pages)
        self._headless = headless

    async def acquire_page(self) -> Page:
        await self._semaphore.acquire()
        browser = await self._get_browser()
        context = await browser.new_context(
            viewport={'width': 1920, 'height': 1080},
            user_agent='Mozilla/5.0 ...',
            locale='zh-CN',
        )
        return await context.new_page()

    async def release_page(self, page: Page):
        await page.context.close()
        self._semaphore.release()
```

## 七、技术栈选择

| 组件 | 选择 | 理由 |
|------|------|------|
| HTTP客户端 | `curl_cffi` | TLS指纹模拟，已验证绕过唯样 |
| 浏览器自动化 | `Playwright` | 比Selenium快，CDP原生支持，已验证5个站点 |
| 反检测 | `playwright-stealth` | 注入反检测JS，减少被识别概率 |
| 验证码 | `ddddocr` | 本地离线，支持滑块+文字，免费 |
| 异步框架 | `asyncio` | 与curl_cffi/Playwright原生兼容 |
| 缓存 | `SQLite (WAL)` | 同BOM1，7天缓存避免重复请求 |
| 代理 | 国内代理池 | 解决地域限制（百能/艾汐/猎芯/京东） |

## 八、与BOM1的对比

| 维度 | BOM1 | 本项目 |
|------|------|--------|
| 站点数 | 2个 (LCSC + HQChip) | 10个 |
| 技术层次 | 纯HTTP (curl_cffi) | HTTP + 浏览器混合 |
| 反爬难度 | 中 (Akamai TLS检测) | 高 (WAF + 验证码 + SPA + 地域限制) |
| 并发模型 | asyncio + Semaphore | asyncio + 浏览器池 + Semaphore |
| 新增依赖 | 无 | Playwright, ddddocr, playwright-stealth |
| 适配器复杂度 | 每个~300行 | HTTP类~200行, 浏览器类~150行, 验证码类~400行 |

## 九、结论

- **可复用BOM1的部分**: 数据模型(PartResult)、适配器工厂(registry)、缓存策略、限速器、查询标准化
- **需新增的部分**: BrowserAdapter基类、浏览器池、验证码模块、API拦截模式
- **核心差异**: BOM1是"纯HTTP能解决所有问题"的场景；本项目是"HTTP解决不了、必须上浏览器"的场景
- **总工期估算**: Phase 1 约 3-4 天，Phase 2 约 2 天，Phase 3 取决于国内环境就绪时间
